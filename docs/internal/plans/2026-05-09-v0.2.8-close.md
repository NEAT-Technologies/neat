## What's NOT in v0.2.8 (gate to next phase)

**The MVP-success-PR experiment (ADR-027).** This is the actual thesis test, not engineering work. Per ADR-027:

> MVP success = closing a real PR on an open-source codebase NEAT was not engineered for. Not running the pg demo. The demo proves the stack works in a controlled environment we built to fail in a specific shape; the MVP earns its keep when NEAT finds a real bug in a real repo, where the OBSERVED layer was load-bearing — not just static analysis a Graphify fork could match.

Every v0.2.x milestone has been preparing the engineering surface for this experiment. v0.2.8 is the last engineering milestone. The next move is:

1. Tag v0.2.8, push, CI publishes (this PR enables that).
2. Pick a target OSS repo per ADR-027 criteria (OTel-friendly, real divergence-shaped bug, responsive maintainers).
3. `npm install -g neat.is@0.2.8`, `neat init <target>`, `neatd start`, run for 24-48 hours under realistic load.
4. Identify a divergence the static-only analysis would miss.
5. Propose a fix, get the upstream PR merged.
