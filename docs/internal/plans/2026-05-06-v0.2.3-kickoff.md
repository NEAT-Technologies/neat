## What v0.2.3 is

The traversal rebuild milestone. Three contracts already on main govern this work:

- **#9 Traversal** ([`docs/contracts/traversal.md`](../contracts/traversal.md), ADR-036) — shared mechanics under both `getRootCause` and `getBlastRadius`. PROV_RANK at every hop. **FRONTIER excluded entirely**, not just deprioritized. Multiplicative confidence cascading. No mutation. Schema-validated results.
- **#10 `getRootCause`** ([`docs/contracts/get-root-cause.md`](../contracts/get-root-cause.md), ADR-037) — origin generality via node-type dispatch (DatabaseNode → driver/engine compat, ServiceNode → node-engine + package-conflict, others → null). Walks incoming to depth 5. Human-readable reason. Fix recommendation derived from compat result.
- **#11 `getBlastRadius`** ([`docs/contracts/get-blast-radius.md`](../contracts/get-blast-radius.md), ADR-038) — BFS outbound, default depth 10. `distance` positive (>=1). Per-node `path` and `confidence` (cascaded). Schema-validated.

The contracts shipped via PR #162. Implementation against them is the work this milestone does.

## Recommended implementation order

1. **#136 — FRONTIER exclusion** (small, ~1 hour). Filter `provenance === FRONTIER` inside `bestEdgeBySource` and `bestEdgeByTarget` in `traverse.ts`. Three live tests flip from `it.todo` (the two in Rule 3, plus the v0.2.3 traversal contract one). FRONTIER currently ranks 0 alongside STALE in `PROV_RANK` — it gets deprioritized but not excluded; the contract requires explicit filtering. New fixture: a graph where the only path between two nodes is via FRONTIER → traversal halts.

2. **#138 — distance positive** (small, ~30 min). One line in `packages/types/src/results.ts` — change `distance: z.number().int().nonnegative()` to `z.number().int().positive()`. Schema shape change in the strict sense, but practical no-op since `traverse.ts:266` already skips frame-0. Run `UPDATE_SNAPSHOT=1` to regenerate the schema snapshot. Flip the `it.todo` to live.

3. **#137 — path + confidence on BlastRadiusAffectedNode** (medium, ~half-day). Schema growth: add `path: z.array(z.string()).min(1)` and `confidence: z.number().min(0).max(1)` to `BlastRadiusAffectedNodeSchema`. The BFS at `traverse.ts:245-287` already tracks parents internally — surface them. Compute `confidence` via `confidenceFromMix` over the edges along the path. Flip three `it.todo` items.

4. **#139 — schema validation** (small, ~30 min). Two `.parse()` calls — one in `getRootCause` before return (line ~232), one in `getBlastRadius` before return (line ~287). The mutation-authority scan already covers `traverse.ts`; this adds the validation guard. Flip four `it.todo` items.

5. **#123 — origin generality** (largest, ~half-day to a day). The dispatch table refactor. Add a `rootCauseShapes` map keyed by `NodeType`:
   - `DatabaseNode` → existing driver/engine compat shape (preserve unchanged).
   - `ServiceNode` → node-engine + package-conflict shapes (call into existing `compat.ts` helpers `checkNodeEngineConstraint`, `checkPackageConflict`).
   - `InfraNode` / `ConfigNode` / `FrontierNode` → return null.
   Refactor `getRootCause` to route through the table. Each shape produces its own `fixRecommendation` string.

Flip 1, 2, 3, 4, 5 in roughly that order. After all five land, all v0.2.3 `it.todo` items in `contracts.test.ts` flip to live; the milestone's contract assertions pass green; tag `v0.2.3` and write release notes.
