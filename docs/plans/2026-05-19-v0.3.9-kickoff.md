## What v0.3.9 is

Three subsystems, one contract batch. Each subsystem closes a v0.3.8 surface gap that became visible once the one-command flow actually shipped:

1. **`neat sync` — the third top-level verb.** `neat <path>` runs the orchestrator once. `neat sync` re-runs the part that matters when the project moves underneath it: rediscover services, re-extract, re-apply the SDK installer for any new entry points, and notify the running daemon. The verb closes the "I added a new package to the workspace — now what?" question that the orchestrator's one-shot shape can't answer.
2. **Env-dimension at ingest.** OTel spans arrive carrying `deployment.environment.name` (or `deployment.environment`, OTel SC compat). Today the ingest pipeline collapses every environment into one graph node — prod and staging traffic from the same `service.name` indistinguishable. Env-dimension adds an `env` discriminator at ingest so the graph carries one node per `(service.name, env)` pair, and the divergence query weights OBSERVED edges within an env separately from cross-env signal.
3. **Framework installer paths — Remix, SvelteKit, Nuxt, Astro.** v0.3.8 added the Next.js dispatch branch to the JS installer (ADR-073 §1). The pattern (`framework:` field on the install plan, framework-specific config edits + entry shape) generalises to four more meta-frameworks that ship with their own runtime-hook surfaces. Each adds one branch to `dispatchEntry` and the `plan`-side dispatcher in `installers/javascript.ts`.

## Why v0.3.9 matters

Two reasons.

1. **The one-command DX surface from v0.3.8 only covers the first 5 minutes.** After the first `neat <path>` run, every project layout change requires the operator to re-reason about which step of the orchestrator to re-run. `neat sync` collapses that decision back into one verb. The first installer paths (vanilla Node, Next.js) cover most demo repos; the four meta-frameworks v0.3.9 adds cover the long tail of real-world targets ADR-027 re-runs will need.
2. **The env-dimension is a precondition for the next ADR-027 re-run.** Any unfamiliar codebase the experiment points NEAT at will likely have a non-trivial prod/staging split. Without env-dimension at ingest, OBSERVED traffic from a staging deploy bleeds into the prod graph and the divergence query's OBSERVED-led weighting (ADR-066) operates on a contaminated set.

## What's in the contract batch (first PR — analogous to #309)

**ADR-074 — `neat sync`, env-dimension at ingest, framework installer paths.** One umbrella ADR with three numbered decisions, matching the v0.3.8 pattern of ADR-073 covering six related surfaces.

- **§1 `neat sync`** — verb shape, what it re-runs vs what it skips, dispatch on daemon-running vs daemon-down, exit codes.
- **§2 env-dimension** — `env` discriminator on `ServiceNode` (and any downstream node whose identity is per-environment), `deployment.environment.name` parsing precedence at ingest, identity-helper change, snapshot migration (per `schema.md`).
- **§3 framework installer paths** — Remix / SvelteKit / Nuxt / Astro dispatch branches, the per-framework config-edit surface, the `framework:` field that downstream consumers read.

Three new per-topic contract files under `docs/contracts/`:

- `contracts/sync.md` — governs `packages/core/src/cli.ts` (sync dispatch) and `packages/core/src/orchestrator.ts` (the re-runnable subset of the orchestrator pipeline).
- `contracts/env-dimension.md` — governs `packages/types/src/identity.ts`, `packages/core/src/ingest.ts`, `packages/core/src/otel.ts`, `packages/core/src/extract/*`, `packages/core/src/persist.ts` (snapshot migration).
- `contracts/framework-installers.md` — governs `packages/core/src/installers/javascript.ts`. Amends — does not supersede — `contracts/sdk-install.md`.

`contracts.test.ts` gains a new `describe('ADR-074 — neat sync + env-dimension + framework installers')` block with three families of assertions (one per section). Pre-implementation rows land as `it.todo`.

Existing contract amendments inside the batch:

- **ADR-031 / `schema.md`** — env-dimension is a shape change (new identity component on ServiceNode). Persist migration writes a one-shot v3 → v4 snapshot upgrade that fills `env: 'unknown'` on every existing node.
- **ADR-028 / `identity.md`** — `serviceId(name, env?)` gains a second optional positional arg. Default unset → `'unknown'` env, preserving the v0.3.8 wire format for any existing snapshot.
- **ADR-066 / `divergence-query.md`** — the OBSERVED-led weighting amendment from v0.3.4 already operates on a per-edge confidence; per-`(service, env)` pair scoping lands as a small clarification, not a new rule.

## Recommended order

The three subsystems are independent — implementation can run in parallel. If sequential, recommended order:

1. **TBD-A (contract batch)** — lands first; nothing implements without the contract.
2. **TBD-C (env-dimension)** — touches the most surfaces (types / identity / ingest / persist) and gates the divergence-query amendment (TBD-H). Land it early so framework installers (TBD-D – TBD-G) can record `framework` + `env` on ServiceNode in one shape.
3. **TBD-D / TBD-E / TBD-F / TBD-G** — four parallelisable installer branches. Each is a single function in `installers/javascript.ts` matching the v0.3.8 Next dispatch shape.
4. **TBD-B (`neat sync`)** — independent of the above; can land anywhere after TBD-A. Useful as a finishing touch because the verb is what surfaces all the other changes to the operator.
5. **TBD-H (divergence weighting tighten)** — lands after TBD-C; small amendment + assertion flip.

## What's not in v0.3.9

- **`neat sync --watch` long-running mode.** v0.3.9 ships the one-shot verb. Continuous re-sync is what `neat watch` + the daemon already cover.
- **`neat deploy` managed-cloud targets.** Railway / Fly / AWS substrate emit branches stay deferred per ADR-073's out-of-scope list.
- **OIDC / mTLS at the daemon boundary.** Bearer auth from ADR-073 remains the only auth shape.
- **Per-env policy scoping.** Policies (ADR-042 — ADR-045) operate on the full graph in v0.3.9. Per-env policy evaluation opens its own ADR if the gap surfaces.
- **The MVP-success-PR experiment re-run (ADR-027).** Gated behind v0.3.9 closing. The env-dimension work is a precondition; the re-run itself is a separate dispatch.
